Control flow and Loops

control flow

Control flow is how the computer reads and executes the code we've written. When we write some instructions the computer will read this from top to bottom. When the computer comes across instructions it will execute the instructions we are giving it in that order. Think of it like watching Netflix. First you open up your computer or turn on your tv, then you select your log in/who is watching Netflix, then we pick something to watch and press play and away we go.

loops

Loops are instructions that will interrupt our control flow and loop through information until we've told it to stop and/or there's no more information to loop through. Once that loop has completed the computer will keep reading and move back through the control flow. Going back to watching Netflix, how often have you opened up Netflix without knowing what you want to watch? The Netflix scroll could be looked at like a loop. Your control flow is interrupted between the logging in and pressing play by scrolling (or looping) through everything there is to watch until you come across what you want to watch and you continue on as per usual.

What is the DOM?

The DOM (Document Object Model) is essentially the family tree of the webpage you're looking at. It is defined by the HTML on the page and we can use it to manipulate all our elements on the page, without having to change the original code. We can also view this in our browser window! We can use the DOM to find different HTML elements or classes through a NodeList. We start off with our Document at the very top. Next we move down to our Nodes. Everything in our document is a node of some description. Our nodes can be Elements, Classes, Attributes etc. If we carry on down the element path underneath that we have our HTML Elements. Our HTML elements will have siblings and children and parents, depending on where they sit in the document. If we think of the body as the parent and next we have an h1 element and underneath that we have 2 p elements, the body element is the parent of the h1 element and the grandparent of the 2 p elements and the 2 p elements are siblings.

The DOM isn't it's own language, it's more like an access point or a gateway into our document. We would use the Javascript language to access the DOM and we can call our results into our console. Let's look at the below screenshot.

I wanted to find where all my h3 elements are on my page. I did this using document.querySelectorAll("h3"). You can see the DOM in action here, it brought up a NodeList, which is like an Array, so I can see what I want to without searching through all my HTML code. It allows me to drop down and see all the information I need to make any necessary changes I want to make.

What the Func(tion)?!

Functions are what are used to control our Javascript. Just as we use style names to control different class styles in CSS, we name our functions in Javascript and use those function names to control and execute our Javascript interactions. Let's look at our example below.

First we have the word function this is telling us that it's a function. Next we have the name of the function iLoveFunctions. We are going to use this when we call the function after we've written our instructions. Next to it we have our parameters (funk, tions). We have 2 parameters in here funk and tions. You can have as many or as few parameters in a function as you need. These hold the input information that the function expects.

After we've named our function and defined our parameters we're going to give our function some instructions. These instructions go in {curly brackets}. In our example we are giving our function an if instruction. We are essentially telling our function that when we call or execute it if parameter 1 is exactly the same as parameter 2 it will return the first string and if it doesn't it will return the second string.Once we've finished with our instructions we have to make sure our whole function is closed off with a } curly bracket.

In order to call our function we write the name of the function iLoveFunctions with what our parameters look like. In this case our parameters are ("Funky", "Funky"). So why do our input parameters look different to what we are calling? Remember our input parameters are information holders. They are just placeholders for the information that we are calling.Let's say we are looking for 2 winners, if we call iLoveFunctions("Winner", "Winner") then yay! We've fulfilled our dream of having 2 winners. If we don't have 2 winners and we call iLoveFunctions("Winner","Loser") then we haven't fulfilled our dream and our 2 parameters don't match.

You can put as many instructions in a function as you like. That's super exciting because it means we can do a whole range of things without having to call each instruction individually!
